Concept: Information Entity
Information entity is a core concept.
The information entity has the attribute ( type: "Fact/Event/Preference", priority: "High" ).
Information entities must enter the long-term memory bank through a memory consolidation process.

Concept: Social Relation
Social relationships describe the connections between different users or entities.
Social relationships are defined as follows: ( relationship_type: "FRIEND_OF/WORKS_WITH/LIKES", timestamp: "Event Time" ).

Concept: Knowledge Update Rule
If new information mentioned by a user conflicts with old information, the new information should be recorded as the latest status quo.
Old information should not be deleted, but should be marked as "historical status".

Concept: Temporal Anchor
Every Event or Fact must be associated with a timestamp.
When comparing two events, the one with a greater distance from the current Session Context is defined as "Happened Earlier".
Logical reasoning must follow the arrow of time: $T_{-2}$ precedes $T_{-1}$.

Concept: Attribute Mapping An Information Entity is not just a name.
It must include attributes such as Quantity, Price, Color, and Specific Designation.
These attributes are "Hard Facts" and must be preserved with 100% accuracy during the consolidation process.

Location: Session Context
The session space is a logical location.
All dialogue, observation, and behavior take place in the conversation space.
The session space evolves with timestamps.
A session space must be connected to the next session space via the NEXT relationship.
If A changes, its associated attribute B should also be re-evaluated.
When there are more than three pieces of evidence from the same source, the confidence level of the belief should automatically increase to 0.9 or higher.
